2. 여행 애플리케이션 - ANDROID, JAVA
 졸업작품으로 제작중인 여행 애플리케이션.
 개발 의도는 여행을 하지 않는 사람들의 대다수가 같이 여행할 동행자가 없다는 것을 보고 동행을 찾는 게시판을 만들어 여행을 활성화하고 이를 통한 여행지 주변의 상권 활성화 및 여행객의 데이터를 나이대별로 종합하여 사용자의 나이대에 맞는 추천지를 제공하여 여행에서의 실망감을 최소화 하고자 제작하였습니다.

2.1. 서버/데이터베이스
 서버는 APM(Apache, PHP, MySQL)을 사용하여 구현하였으나 Go언어를 배우고 있는 과정에서 PHP파일을 Go언어로 변경하여 구현해보고자 하여 Go언어로도 구현되어 있는 상태입니다.
 Go언어는 Apache 없이 내장 모듈을 사용하여 구동이 가능하기에 APM으로 구현한 서버 포트는 :80 즉, 기본포트로 설정하였고 Go언어는 :9090 포트로 설정하였습니다.

 데이터베이스는 MySQL을 사용하였습니다. 데이터 관리에는 phpmyadmin과 MySQL Workbench를 사용하였습니다.

2.2. Firebase Cloud Messaging(FCM)
 안드로이드에서 푸시 메시지 알림을 받기 위한 Firebase API로서 구글에서 GCM 보다는 FCM을 사용하라고 권고함에 따라 FCM으로 구현하였습니다.
 FCM으로 구현된 기능은 내가 선택한 지역의 게시글이 등록될 때 해당 지역의 게시글이 등록되었다고 푸시 알림을 줍니다. 이외에도 구현은 하지 않았으나 전체 알림 메시지와 같은 곳에서 사용할 수 있습니다.

2.3. 실행화면

↑ 주어진 쿼리문을 실행하고 나온 값과 인증 파일을 통해 FCM에 푸시 메시지를 요청


↑ 구글맵 마커를 커스텀하여 해당 위치에 마커 대신 해당지역 이미지를 띄우고 클릭 시 해당 지역 내용을 출력

8. 인사관리 프로그램 - JAVA
 디자인 패턴과 JAVA를 통하여 인사관리 프로그램을 제작하였습니다.

 구현에 사용된 디자인 패턴은 스트래티지 패턴, 이터레이터 패턴, 프록시 패턴, 메멘토 패턴, 커맨드 패턴 입니다.

8.1. 디자인 패턴
 스트래티지 패턴 : 기능 군을 정의, 캡슐화 하고 교환을 통해 전략을 동적으로 쉽고 자유롭게 바꿈으로써 유연성 증가. 디자인 원칙 중 분리, 상속 보다는 구성을 활용하라는 원칙을 적용한 패턴.

 이터레이터 패턴 : 반복수행에 대해 반복문을 사용하지 않고 반복자 객체에게 일임하여 책임을 분리. 인터페이스 및 구현이 간단해지고 집합체는 객체 컬렉션 관리에 전념.

 원격 프록시 패턴 : 프록시의 메소드를 호출하면 네트워크를 통해서 전달되어 결국 원격 객체의 메소드가 호출. 결과는 다시 프록시를 거쳐서 클라이언트에 전달.

 메멘토 패턴 : 객체의 상태 정보를 저장 및 복원하는 패턴. 객체의 내부 상태 정보만 가지는 클래스를 따로 생성하여 관리하는 구조.

 커맨드 패턴 : 요구사항을 객체로 캡슐화 할 수 있고 매개변수를 써서 여러가지 다른 요구사항 삽입 가능. 커맨드 객체는 수신자 객체를 가지고 있으며 수신자의 메서드를 호출하고 수신자는 정의된 메서드를 수행.
